Sustech ECU Telemetry Reverse Engineering – Technical Summary

1. Objective

The objective of this exercise was to understand and decode the live serial telemetry stream from the Sustech ECU in order to:

Extract real-time engine data without the official tuning software
Integrate ECU data into a custom digital dashboard
Validate sensor scaling and packet structure
Build a reliable external decoder for telemetry
2. Data Capture & Initial Observations

Raw serial data was captured directly from the ECU at:

Baud rate: 115200
Protocol: Continuous stream

Initial inspection showed repeating structured data blocks.

3. Packet Framing Identification

A consistent header was identified:

83 84 65 82 84

ASCII:

S T A R T
Conclusion:

The ECU transmits structured packets beginning with a "START" header.

4. Packet Structure

Through analysis, the packet length was determined to be approximately:

120 bytes per frame

The ECU transmits at a fixed rate of:

25 samples per second (40 ms interval)
Conclusion:
Fixed-length packets
Fixed-rate streaming (time-based, not event-based)
5. Packet Synchronization Method

Reliable decoding required:

Searching for "START" header
Aligning reads to full packet length (120 bytes)

Without synchronization, data appeared corrupted (jumping values, incorrect RPM, etc.).

6. Identified Data Fields

The following byte positions have been confirmed:

Byte Index	Function
0–4	Packet header ("START")
6	RPM
13	MAP / Boost
14	TPS
16	Coolant Temp
17	Coolant Status
18	Intake Air Temp
19	IAT Status
20–21	Battery Voltage
28	GPO1 Output

7. Key Discovery: Data Type Behavior
Initial assumption:
Sensors were raw ADC or resistance values
Actual finding:
ECU transmits pre-scaled linear values, not raw ADC

This was determined by:

Comparing raw bytes vs ECU debug screen values
Observing linear relationships rather than thermistor curves
8. Sensor Decoding Methodology
8.1 Linear Calibration Approach

For each sensor:

Capture known ECU value from Sustech software
Record corresponding raw byte
Use 2-point linear interpolation:
y = A + Bx

Where:

x = raw byte
y = actual sensor value
9. Final Sensor Calibrations
9.1 RPM

Measured:

raw 35  = 930 RPM
raw 111 = 2500 RPM

Formula:

RPM = 207 + (20.66 × raw)
9.2 Boost / MAP

Measured:

raw 104 = -0.176 bar
raw 155 = -0.639 bar

Formula:

Boost (bar) = 0.767 - (0.00907 × raw)

Additional small correction applied (~0.1 bar).

9.3 Coolant Temperature (CLT)

Measured:

raw 167 = 75.1°C
raw 96  = 81.9°C

Formula:

CLT (°C) = 91.1 - (0.0958 × raw)
9.4 Intake Air Temperature (IAT)

Measured:

raw 225 = 33.9°C
raw 102 = 49.5°C

Formula:

IAT (°C) = 62.43 - (0.1268 × raw)
9.5 Battery Voltage
Battery = raw / 64
9.6 TPS
TPS = packet[14] (direct byte scaling)
9.7 GPO Output
Byte 28:
0   = OFF
255 = ON

10. Behavioural Observations
Temperature sensors:
Raw value decreases as temperature increases
Indicates NTC origin, but transmitted as linearized value
Boost:
Raw value increases → more vacuum (more negative pressure)
RPM:
Linear single-byte representation (scaled)
11. Decoder Implementation

A Python-based live decoder was built featuring:

Serial packet synchronization
Real-time decoding (25 Hz)
Dual display:
Decoded values
Raw packet values
Stable, non-scrolling console output

14. Conclusions
The Sustech ECU uses a structured, fixed-rate telemetry protocol
Data is transmitted as pre-scaled linear values, not raw ADC
Packet structure is consistent and predictable
External decoding is fully feasible without proprietary software
